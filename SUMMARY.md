# Phase 1 — Scaffold (summary)

## What landed

- **Next.js 14.2** app (`stock-seasonality/`) with App Router, TypeScript, Tailwind CSS, ESLint, src/ layout, `@/*` import alias.
- **Prettier 3** with `prettier-plugin-tailwindcss` for auto-sorting class names. `.prettierrc`, `.prettierignore`, `npm run format` / `format:check`.
- **Prisma 7** initialized for PostgreSQL (`prisma init --datasource-provider postgresql`). Empty schema — real models land in Phase 2. `dotenv` wired in via `prisma.config.ts` so `.env` is loaded for CLI commands.
- **`.env.example`** committed with placeholders for Supabase (`DATABASE_URL` pooled, `DIRECT_URL` direct) plus `NEXT_PUBLIC_APP_URL`. `.env` is gitignored; `.env.example` is the template.
- **Layout shell**: sticky top nav, max-w-6xl content container, footer with "Not investment advice" disclaimer baked in from day 1. Geist Sans / Geist Mono fonts. Light + dark mode via `prefers-color-scheme`.
- **Routing skeleton** for every Phase 1 path:
  - `/` landing page with the three entry points (by category / by stock / by event)
  - `/categories` placeholder grid
  - `/category/[slug]` placeholder detail page
  - `/stock/[ticker]` placeholder detail page
  - `/calendar` placeholder events list
  - `/about` methodology + disclaimers stub
- Per-page `metadata` exports (title template `%s · Stock Seasonality Analyzer`, descriptions).

## Notes for review

- **Prisma 7 (not 5).** `npm i prisma` pulled `^7.8.0`. The generator emits the client to `src/generated/prisma` (gitignored, regenerated by `prisma generate`). If you'd rather pin to Prisma 5.x for stability, say the word and I'll downgrade before Phase 2 schema work.
- **Tailwind v3.** Default from `create-next-app@14`. Tailwind v4 would mean a different setup (CSS-first config); sticking with v3 keeps Phase 6 SEO/typography plugins straightforward.
- **No `prose` typography plugin yet** — the About page uses `prose` class names but the `@tailwindcss/typography` plugin isn't installed. The page still renders fine (Tailwind just ignores the unknown classes). I'll add it in Phase 6 when we polish the About page properly. Easy to pull forward if you want.
- **Disclaimer footer is global.** Phase 6 calls for it on "every analysis page" but it's cheap to apply globally, and it sets the tone from page 1.
- **`globals.css` body font** swapped from the default Arial fallback to `var(--font-geist-sans)` so the Geist font actually applies. Small but the default scaffold leaves this misconfigured.
